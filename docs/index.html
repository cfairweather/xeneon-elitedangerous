Xeneon Elite Dangerous HUD

A real-time companion widget for the Corsair Xeneon Edge 14.5" touchscreen display — live ship telemetry, navigation, cargo manifest, and mission tracking, all in one tap.

Tab Reference

Tab 1

HUD

At-a-glance ship telemetry: hull, shields, fuel, cargo bars, power pips, location, game mode, legal status, credits, and tactical flags (hardpoints, silent running, FSD, etc.).

Tab 2

NAV

Current location with body and station name. Plotted route with per-system star class colour-coding, jump count, and destination. Scrollable for long routes (Colonia runs, Beagle Point, etc.).

Tab 3

CARGO

Full cargo manifest sorted alphabetically. Each line shows commodity, quantity in tonnes, and tags for STOLEN and MISSION cargo. Shows current / max capacity.

Tab 4

MISSIONS (#)

All active missions with destination, credit reward, and live countdown timer to expiry. Sorted by soonest expiry first. The orange badge on the tab shows how many missions are active.

Setup Guide

What you need Corsair Xeneon Edge (14.5" LCD, 2560×720) · iCUE 5 or later · Node.js 18+ · Windows 10/11

Step 1 — Get the files

Clone or download the repository:

git clone https://github.com/cfairweather/xeneon-elitedangerous.git

Or download the ZIP from GitHub and extract it anywhere you like.

Step 2 — Install the widget in iCUE

  1. Open iCUE

    Launch the Corsair iCUE application. Make sure your Xeneon Edge is connected and appears in the device list.

  2. Open Widget Builder

    Click on the Xeneon Edge in iCUE, then select Widget Builder from the controls that appear.

  3. Import the widget package

    Drag the com.fairweather.elitedangerous/ folder into the Widget Builder, or use File → Import Widget. The widget appears in your library as "Elite Dangerous HUD".

  4. Assign to the XL-H slot

    Drag the widget onto the XL-H slot (2536×696) on your Xeneon Edge canvas. This is the full-width slot that fills the screen — all four tabs and the tactical panel are visible here.

Step 3 — Start the companion server

Why is this needed? iCUE widgets run as sandboxed web pages and cannot access the filesystem directly. The companion server is a small Node.js process that runs outside the sandbox, reads your Elite Dangerous journal files, and streams events to the widget over a local WebSocket.

  1. Install Node.js (first time only)

    Download and install Node.js 18 LTS or later from nodejs.org. The installer adds node and npm to your PATH automatically.

  2. Launch the server

    Double-click companion\start.bat in the repository folder. The first run installs the single dependency (ws). Keep the window open while playing.

    companion\start.bat

    You should see:

    [server] Elite Dangerous Journal Server
[server] ws://localhost:31337
[server] Journal dir: C:\Users\...\Saved Games\...

  3. Or start manually

    cd companion
npm install      # first time only
npm start
Security note The server binds to 127.0.0.1:31337 only — it is not reachable from other computers on your network.

Step 4 — Launch Elite Dangerous

Start the game. The widget shows "Connecting to GalNet" with an animated scan bar until the companion server is running. Once the server is up and the game emits its first journal events, the HUD populates automatically.

If the connection drops mid-session, the widget shows "GalNet Signal Lost" and retains the last known data. It reconnects automatically with exponential backoff (max 30 second interval).

Startup order

Any order works — the widget will wait and reconnect. But for the smoothest start:

  1. Start companion\start.bat
  2. Launch iCUE (or confirm it's already running)
  3. Launch Elite Dangerous

Architecture

Elite Dangerous (game process)
│ writes Journal.*.log + Status.json every ~1 s
▼
companion/server.js (Node.js — run separately)
│ tails files, broadcasts events
│ ws://127.0.0.1:31337
▼
iCUE widget (Chromium sandbox — QtWebEngine)
│ WebSocket client, state machine, tab renderer
▼
Corsair Xeneon Edge (14.5" touchscreen, 2560×720)

Journal events consumed

EventWhat the widget learns

LocationStar system, body, station
FSDJumpNew system, fuel level, route advancement
NavRouteFull plotted route (NAV tab list)
NavRouteClearClears route display
Docked / UndockedStation name
LoadoutShip details, max jump range, cargo capacity
HullDamageHull health
ShieldStateShields up/down
Cargo (with Inventory)Full cargo manifest
CollectCargo / EjectCargoCargo delta updates
MarketBuy / MarketSellCargo delta updates
MissionAcceptedAdd mission with reward & expiry
MissionCompleted/Failed/AbandonedRemove mission
Status (NEW_STATUS_EVENT)Flags, fuel, cargo, pips, legal state

Companion Server

companion/server.js is ~200 lines of vanilla Node.js with a single npm dependency (ws for WebSocket). It requires no config and has no cloud or telemetry components.

BehaviourDetail
Journal pollingEvery 500 ms — tails the latest Journal.*.log from byte offset; never re-reads old data
Status pollingEvery 1000 ms — checks Status.json mtime before reading
Replay bufferLast 300 journal events sent to each new connection so the widget rebuilds full state instantly
Port conflictExits with a clear error if port 31337 is already in use
Graceful shutdownSIGINT / SIGTERM (Ctrl+C or Windows close button)

Message format

Compatible with the elite-dangerous-journal-server convention:

{ "type": "NEW_EVENT",        "payload": { "event": "Docked", ... } }
{ "type": "NEW_STATUS_EVENT", "payload": { "Flags": 12345, "Fuel": {...}, ... } }

Responsive Layout

The HUD tab adapts to four slot sizes using CSS aspect-ratio and min-height media queries — no JavaScript class toggling.

SlotApprox. sizeHUD panels shown
XL-H2536×696Location, Vessel, Vitals, Tactical
L-H~1800×696Location, Vessel, Vitals (Tactical hidden)
M-H~840×696Location, Vitals (Vessel + Tactical hidden)
S-H~800×400Location, Vitals (header + labels stripped)

The NAV, CARGO, and MISSIONS tabs are single-column scrollable layouts and work in any slot size.

Regenerating Screenshots

After making UI changes, regenerate the preview images with Playwright:

pip3 install playwright
python3 -m playwright install chromium

# From the repo root — start a local server first:
python3 -m http.server 5500 --directory com.fairweather.elitedangerous &
python3 scripts/screenshot.py

Screenshots are written to docs/images/.
